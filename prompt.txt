Architecture Principle — Separation of Concerns
Every page loads only what it needs. The loading order on every HTML file must be:
html<!-- 1. Shared base styles -->
<link rel="stylesheet" href="/css/base.css">
<link rel="stylesheet" href="/css/layout.css">
<link rel="stylesheet" href="/css/components.css">

<!-- 2. Page-specific styles -->
<link rel="stylesheet" href="/css/home.css">   <!-- only on index.html -->

<!-- 3. Shared config (always first JS) -->
<script src="/js/config.js"></script>

<!-- 4. Page-specific data + logic -->
<script src="/js/home.js" defer></script>

<!-- 5. Shared nav (always last) -->
<script src="/js/nav.js" defer></script>
If you need to change the hero headline on the home page, you edit data/home.json only. If you need to change card layout for IT service detail pages, you edit css/it-service.css only. No hunting.

Navigation — Two Separate Systems
Mobile (< 768px):

Persistent bottom navigation bar on every page: Home, IT Services, Marketing, About, Contact
Active tab highlighted based on current URL path — handled entirely by nav.js
Hub pages serve as landing menus — service cards link to detail pages
No hamburger menu; bottom nav is the only global nav
Bottom nav markup lives in every HTML file (or injected by nav.js)

Desktop (≥ 768px):

Sticky top horizontal nav bar
"IT Services" and "Marketing" are CSS-first dropdown menus (JS toggle as fallback), listing all child pages
Bottom nav hidden via layout.css media query — no JS required to hide it
Logo left, nav links right
Dropdown link list is generated by nav.js reading from it-services.json and marketing-services.json so adding a new service auto-adds it to the dropdown

Technical Requirements

All internal links are proper href paths — zero JS page switching
Back buttons on detail pages link to ../it/ or ../marketing/
No external CSS frameworks or JS libraries — vanilla only
Valid HTML5, proper <meta> tags on every page (charset, viewport, description, og:title, og:description)
Changing copy: edit the relevant JSON file only
Changing a colour: edit base.css only
Changing a layout: edit the relevant CSS file only
Changing business logic: edit the relevant JS file only
